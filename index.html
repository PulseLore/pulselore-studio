Privacy / Policy / Security

Clear rules. Safer work.

PulseLore Studio keeps public pages honest: demos are previews, private tools stay private, and creator services are handled with rights and permission boundaries.

Privacy

Contact form details are used only to reply to the requested project, service, demo, or portfolio inquiry. Private client materials are not published without permission.

Policy

Public demos show the direction and scope of a tool. Final features, pricing, hosting, payment methods, and maintenance are confirmed before paid work begins.

Security

API keys, passwords, private prompts, admin notes, and access codes must not be placed in public frontend files. Private tools use locked access or backend protection where needed.

Rights

RVC, cover songs, artist models, distribution support, and release preparation require proper permission. PulseLore does not support unauthorized copy, cover, or artist model training work.

Data Handling

Browser-only tools such as Music Meter process files locally when stated. If a future tool needs upload, database, or AI API processing, that will be clearly disclosed first.

Service Terms

Each paid project should have a written scope, payment schedule, delivery expectation, and maintenance agreement. New feature requests outside scope are estimated separately.
