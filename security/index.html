Security White Paper

PulseLore AI Music Meter — Beta. Effective Date: [insert date]. Version 0.1 Beta.

Privacy-first storage model

Beta preview status: real audio upload and temporary storage are not enabled yet. Planned model: uploaded audio should be stored temporarily only for analysis, and the original file must be deleted manually on request or automatically within 24 hours.

Training boundary

Planned backend rule: uploaded audio must not be used for AI training. It must not be sold, published, shared publicly, or distributed by PulseLore.Studio.

Future database fields

Planned fields include user_id, email, role, monthly_limit, used_count, upload_id, song_title, artist_name, upload_time, delete_deadline, deleted_at, consent_checked, policy_version, report_score, and report_text.

Audit log

The MVP should log uploaded, analyzed, manually deleted, and auto-deleted events. The auto-delete job should run at least once per hour.

Security status

This prototype is UI-only. Google login, Supabase database, temporary audio storage, Python audio analyzer, 24-hour auto-delete cron job, quota enforcement, upload deletion, and audit logging are not connected yet.
